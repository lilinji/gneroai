## 🎯 平台基本信息

### 系统环境
- **操作系统**: 64位 RedHat 8.8
- **协议支持**: SSH、SCP、FTP
- **登录域名**: `login.tibhpc.net`
- **端口**: 22（SSH默认端口）

### 访问要求
- 拥有有效的平台账户
- 支持SSH协议的客户端软件
- 稳定的网络连接

## 🔗 登录方式概览

平台支持两种主要登录方式：

| 登录方式 | 适用场景 | 网络要求 | 推荐工具 |
|---------|---------|---------|---------|
| **所内直接登录** | 校内办公、实验室使用 | 连接TIB有线/无线网络 | Xshell、MobaXterm、Terminal |
| **所外VPN登录** | 居家办公、出差访问 | 公网+VPN连接 | aTrust VPN + SSH客户端 |

## ⚙️ SSH客户端配置

### Linux/macOS Terminal

```bash
# 基本SSH连接
ssh username@login.tibhpc.net

# 指定端口连接
ssh -p 22 username@login.tibhpc.net

# 启用X11转发（图形界面应用）
ssh -X username@login.tibhpc.net
```

### Windows PowerShell

```powershell
# Windows 10/11内置SSH客户端
ssh username@login.tibhpc.net
```

### SSH密钥配置（推荐）

为提高安全性和使用便利性，建议配置SSH密钥认证：

#### 生成密钥对

```bash
# 生成RSA密钥对
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

# 查看公钥内容
cat ~/.ssh/id_rsa.pub
```

#### 上传公钥

```bash
# 方法一：使用ssh-copy-id（推荐）
ssh-copy-id username@login.tibhpc.net

# 方法二：手动复制
scp ~/.ssh/id_rsa.pub username@login.tibhpc.net:~/
```

#### 服务器端配置

```bash
# 登录服务器后执行
mkdir -p ~/.ssh
cat ~/id_rsa.pub >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
chmod 700 ~/.ssh
rm ~/id_rsa.pub
```

## 🚨 常见问题解决

### 问题1：连接超时
**症状**: Connection timeout 或 Network is unreachable

**解决方案**:
1. 检查网络连接状态
2. 确认是否在TIB网络内（所内登录）
3. 验证VPN连接状态（所外登录）
4. 尝试使用备用VPN地址

### 问题2：认证失败
**症状**: Authentication failed 或 Permission denied

**解决方案**:
1. 确认用户名和密码正确
2. 检查账户是否被锁定
3. 联系管理员重置密码
4. 验证认证方式设置

### 问题3：主机密钥错误
**症状**: Host key verification failed

**解决方案**:
```bash
# 删除旧的主机密钥
ssh-keygen -R login.tibhpc.net

# 重新连接并接受新密钥
ssh username@login.tibhpc.net
```

### 问题4：VPN无法连接
**症状**: VPN连接失败或断开

**解决方案**:
1. 检查网络防火墙设置
2. 尝试不同的VPN服务器地址
3. 重启VPN客户端
4. 联系网络管理员

## 🔒 安全最佳实践

### 密码安全
- ✅ 使用强密码（包含大小写字母、数字、特殊字符）
- ✅ 定期更换密码（建议3-6个月）
- ✅ 不要在多个系统使用相同密码
- ❌ 不要在公共场所输入密码
- ❌ 不要将密码写在纸上或保存在不安全的地方

### 连接安全
- ✅ 使用SSH密钥认证替代密码认证
- ✅ 在安全网络环境下进行登录
- ✅ 及时退出登录会话
- ✅ 定期检查登录日志
- ❌ 避免在公共WiFi下直接登录

### 账户管理
- ✅ 不要共享个人账户
- ✅ 发现异常登录及时报告
- ✅ 离职或账户不再使用时及时申请注销
- ❌ 不要让他人使用您的账户
